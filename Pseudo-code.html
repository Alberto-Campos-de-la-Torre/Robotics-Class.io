¿Cómo le Enseñas a una Máquina?

Imagina que tienes que explicarle a alguien cómo hacer un sándwich, pero esa persona solo puede seguir instrucciones literales y no puede improvisar. Tendrías que darle una lista de pasos muy precisa y ordenada. La programación es exactamente eso: crear una "receta" perfecta para que una computadora o un robot la siga.

Planificando Antes de Programar

Algoritmo

Un algoritmo es simplemente una serie de pasos finitos y ordenados para resolver un problema o completar una tarea. Es la "receta". Antes de escribir código, siempre debemos tener claro nuestro algoritmo.

Pseudocódigo

Es una forma de escribir un algoritmo usando un lenguaje simple y parecido al humano, pero con una estructura que se asemeja a la de un lenguaje de programación. No se puede compilar, pero es perfecto para planificar.

Diagrama de Flujo

Es la representación gráfica de un algoritmo. Usa símbolos estandarizados para mostrar los pasos y las decisiones, lo que lo hace muy fácil de seguir visualmente. Es el "mapa" de nuestra receta.

Los Símbolos de un Diagrama de Flujo

Terminal

Representa el Inicio y el Fin del algoritmo. Siempre hay uno de cada uno.

Proceso

Indica una acción o una operación, como un cálculo matemático o una asignación de valor.

Entrada/Salida

Muestra dónde el algoritmo recibe datos (LEER) o muestra resultados (ESCRIBIR).

Decisión

Representa una pregunta (normalmente sí/no) que divide el flujo del programa en dos o más caminos.

Estructura Básica del Pseudocódigo

El pseudocódigo nos ayuda a estructurar nuestro pensamiento. Aunque no hay una regla estricta, solemos usar palabras clave para definir acciones comunes.

INICIO
  // Las variables guardan información
  Definir numero1, numero2, suma Como Entero

  // Pedimos datos al usuario
  ESCRIBIR "Dame el primer número:"
  LEER numero1

  ESCRIBIR "Dame el segundo número:"
  LEER numero2

  // Realizamos una operación
  suma = numero1 + numero2

  // Tomamos una decisión
  SI suma > 10 ENTONCES
    ESCRIBIR "El resultado es mayor que 10."
  SINO
    ESCRIBIR "El resultado es 10 o menor."
  FIN SI

  // Mostramos el resultado
  ESCRIBIR "La suma es: ", suma
FIN

Actividades de Lógica

Actividad 1: De Diagrama a Pseudocódigo

Observa el siguiente diagrama de flujo, que diagnostica por qué una lámpara no funciona. Traduce esta lógica a pseudocódigo.

INICIO
  Definir esta_enchufada, foco_quemado Como Logico

  ESCRIBIR "¿La lámpara está enchufada? (v/f)"
  LEER esta_enchufada

  SI esta_enchufada == falso ENTONCES
    ESCRIBIR "Enchufar la lámpara."
  SINO
    ESCRIBIR "¿El foco está quemado? (v/f)"
    LEER foco_quemado

    SI foco_quemado == verdadero ENTONCES
      ESCRIBIR "Reemplazar el foco."
    SINO
      ESCRIBIR "Comprar una nueva lámpara."
    FIN SI
  FIN SI
FIN

Actividad 2: Diseña un Algoritmo

Escribe el pseudocódigo para el sistema de control de una cafetera simple. Considera las siguientes reglas:

  • La cafetera necesita que haya agua en el depósito para funcionar.
  • La cafetera necesita que haya café en el filtro.
  • Si ambas condiciones se cumplen, y el usuario presiona el botón de "Iniciar", la cafetera debe encender la resistencia para calentar el agua.
  • Si alguna de las dos primeras condiciones no se cumple al presionar "Iniciar", debe encender una luz roja de "Error".
INICIO
  Definir hay_agua, hay_cafe, boton_iniciar Como Logico

  // Simula la lectura de sensores y botones
  LEER hay_agua
  LEER hay_cafe
  LEER boton_iniciar

  SI boton_iniciar == verdadero ENTONCES
    SI hay_agua == verdadero Y hay_cafe == verdadero ENTONCES
      ESCRIBIR "Encender resistencia."
    SINO
      ESCRIBIR "Encender luz de error."
    FIN SI
  FIN SI
FIN

Conectando la Lógica con los Robots

El pseudocódigo y los diagramas de flujo son nuestros planos. Una vez que tenemos un plan sólido que funciona en papel, el siguiente paso es traducirlo a un lenguaje que un microcontrolador (como Arduino o ESP32) pueda entender y ejecutar. Las estructuras SI-ENTONCES y los bucles MIENTRAS que planeamos en pseudocódigo se convertirán en if-else y while en lenguajes como C++.
