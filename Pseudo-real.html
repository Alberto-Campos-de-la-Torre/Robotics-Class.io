El Puente entre Pseudocódigo y Código Real

La planificación que hacemos con pseudocódigo no es un ejercicio teórico, es el plano exacto de nuestro futuro programa. Observa cómo un algoritmo simple para hacer parpadear un LED se traduce directamente a C++, el lenguaje que usa Arduino.

1. El Plan (Pseudocódigo)

INICIO   // Configuración inicial   Definir el pin del LED (pin 13) como SALIDA   // Bucle principal que se repite por siempre   MIENTRAS verdadero HACER     // Encender el LED     Poner el pin del LED en ALTO     // Esperar un momento     Esperar 1 segundo     // Apagar el LED     Poner el pin del LED en BAJO     // Esperar otro momento     Esperar 1 segundo   FIN MIENTRAS FIN

2. La Ejecución (Código Arduino C++)

// Configuración inicial void setup() {   // Definir el pin del LED (pin 13) como SALIDA   pinMode(13, OUTPUT); } // Bucle principal que se repite por siempre void loop() {   // Encender el LED   digitalWrite(13, HIGH);   // Esperar un momento   delay(1000); // 1000 milisegundos = 1 segundo   // Apagar el LED   digitalWrite(13, LOW);   // Esperar otro momento   delay(1000); }

Repitiendo Tareas: Los Bucles

En robótica, casi siempre queremos que las acciones se repitan. Los bucles son las estructuras que nos permiten hacer esto sin tener que escribir la misma línea de código una y otra vez.

Bucle MIENTRAS (While)

Se ejecuta mientras una condición sea verdadera. Es ideal cuando no sabemos exactamente cuántas veces necesitamos repetir la acción.

INICIO   Definir contador Como Entero = 0   MIENTRAS contador < 5 HACER     ESCRIBIR "El contador es: ", contador     contador = contador + 1   FIN MIENTRAS FIN

Bucle PARA (For)

Se ejecuta un número específico de veces. Es perfecto cuando sabemos de antemano cuántas repeticiones queremos.

INICIO   PARA i = 1 HASTA 3 HACER     ESCRIBIR "Esta es la repetición número ", i   FIN PARA FIN
